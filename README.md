# Customer Churn Prediction

An end-to-end machine learning project that predicts whether a telecom customer
will churn, built with **Python, Pandas, and Scikit-learn**.

## Project Structure

```
churn_project/
├── data/
│   ├── generate_data.py       # Generates a realistic synthetic dataset (7,000 customers)
│   └── customer_churn.csv     # The dataset
├── churn_prediction.py        # Main pipeline: EDA -> preprocessing -> training -> evaluation
├── predict_new_customer.py    # Example script to score new/unseen customers
├── models/
│   └── churn_model.joblib     # Saved best-performing trained pipeline
├── outputs/                   # Charts & metrics generated by the pipeline
└── requirements.txt
```

## Dataset

No dataset was provided, so this project uses a **synthetic but realistic**
telecom customer dataset (`data/generate_data.py`) modeled on the well-known
Telco Customer Churn dataset schema. Churn is driven by genuine underlying
patterns — contract type, tenure, monthly charges, support tickets, add-on
services, etc. — plus noise, so the models have real signal to learn and the
results are meaningful.

**To use your own data instead:** replace `data/customer_churn.csv` with your
file (keep a `Churn` column with Yes/No values), and adjust column names in
`churn_prediction.py` if they differ.

Features include:
- Demographics: gender, SeniorCitizen, Partner, Dependents
- Account info: tenure, Contract, PaperlessBilling, PaymentMethod
- Services: PhoneService, MultipleLines, InternetService, OnlineSecurity,
  OnlineBackup, DeviceProtection, TechSupport, StreamingTV, StreamingMovies
- Billing: MonthlyCharges, TotalCharges
- Engagement signal: SupportTickets

## How to Run

```bash
pip install -r requirements.txt

# 1. Generate the dataset (optional — already included)
python3 data/generate_data.py

# 2. Run the full training pipeline
python3 churn_prediction.py

# 3. Try predictions on new customers
python3 predict_new_customer.py
```

## Pipeline Steps (`churn_prediction.py`)

1. **Load & Clean** — drop non-predictive ID column, encode target, fix dtypes.
2. **EDA** — churn distribution, churn rate by contract type, tenure vs churn,
   monthly charges vs churn, correlation heatmap. Saved as PNGs in `outputs/`.
3. **Preprocessing** — `ColumnTransformer` with:
   - Numeric features: median imputation + `StandardScaler`
   - Categorical features: most-frequent imputation + `OneHotEncoder`
4. **Model Training & Comparison** — three models trained and compared via
   5-fold cross-validated ROC-AUC plus held-out test metrics:
   - Logistic Regression
   - Random Forest
   - Gradient Boosting
5. **Evaluation** — accuracy, precision, recall, F1, ROC-AUC, confusion
   matrix, ROC curves.
6. **Feature Importance** — top drivers of churn (coefficients or
   `feature_importances_` depending on the winning model).
7. **Model Saving** — best model (by test ROC-AUC) is saved to
   `models/churn_model.joblib` as a full scikit-learn `Pipeline` (preprocessing
   + classifier bundled together, so it can be applied directly to raw new data).

**Best model: Logistic Regression** (ROC-AUC 0.81), selected automatically by
the script based on test-set ROC-AUC.

Top churn drivers identified: **contract type** (month-to-month = high risk,
two-year = low risk), **tenure** (shorter tenure = higher risk), **monthly
charges**, **total charges**, and **lack of tech support / online security**.

Since results depend on random data generation, exact numbers may vary slightly
if you regenerate the dataset.

## Using the Saved Model

```python
import joblib
import pandas as pd

model = joblib.load("models/churn_model.joblib")

new_customer = pd.DataFrame([{
    "gender": "Female", "SeniorCitizen": 0, "Partner": "No", "Dependents": "No",
    "tenure": 2, "PhoneService": "Yes", "MultipleLines": "No",
    "InternetService": "Fiber optic", "OnlineSecurity": "No", "OnlineBackup": "No",
    "DeviceProtection": "No", "TechSupport": "No", "StreamingTV": "Yes",
    "StreamingMovies": "Yes", "Contract": "Month-to-month", "PaperlessBilling": "Yes",
    "PaymentMethod": "Electronic check", "MonthlyCharges": 95.5, "TotalCharges": 191.0,
    "SupportTickets": 4,
}])

prob = model.predict_proba(new_customer)[:, 1][0]
print(f"Churn probability: {prob:.1%}")
```
